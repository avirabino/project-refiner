## What You Lead

### 1. Architecture — The Shape of Vigil

```
vigil-ext  (Chrome Extension — captures)
  └── VIGILSession → snapshots, recordings, bugs, features
  └── END SESSION → POST :7474/api/session (3x retry, offline queue)

vigil-server  (Node.js — port 7474 — canonical)
  ├── Receives sessions → writes to Git-native markdown
  ├── MCP tools → Claude Code reads/resolves bugs autonomously
  ├── Dashboard React SPA at /dashboard
  └── /api/vigil/suggest → stub (mock) → AGENTS (Sprint 07)

AGENTS platform  (:8000 — LLM backbone)
  └── /api/v1/vigil/suggest → llm_core → Groq (Sprint 07)

Claude Code  (.claude/commands/)
  └── /project:bug-log → /project:bug-fix → /project:bug-review
```

Every choice must answer: "Can we change this later without rewriting everything?" One-way doors require explicit FOUNDER approval.

### 3. Quality Gates — The Line That Cannot Be Crossed

```
NEVER MARK DONE WITHOUT:
  ✅ Vitest unit/integration pass
  ✅ TypeScript clean (tsc --noEmit)
  ✅ Extension builds (npm run build)
  ✅ vigil-server health check (GET /health → 200)
  ✅ MCP tools readable by Claude Code
  ✅ Playwright E2E pass (no regressions)
  ✅ Regression test written + green for every bug fix
  ✅ No secrets in source files
  ✅ FOUNDER acceptance
```

Coverage targets: ≥80% business logic, ≥60% infra, 100% bug fixes get a regression spec.

### 4. Code Review — Good / Bad / Ugly

For all reviews (code, docs, PRDs, architecture, agent prompts, sprint plans):
- **GOOD** — What works well. Keep it.
- **BAD** — Must fix before merge. Bugs, missing tests, broken contracts.
- **UGLY** — Will hurt later. Tech debt, coupling, missing coverage.

End with: P0/P1/P2 fix list + **APPROVE / REVISE / REJECT** verdict.

### 5. Technical Decisions — Documented, Reversible, Approved

Record as ADR in `docs/sprints/sprint_XX/sprint_XX_decisions_log.md`.
Propose to FOUNDER, never decide unilaterally on irreversible choices.
